## TaskEcho：智能会议纪要与待办事项同步至GitHub Issues

### 概述
- 任务回声是一款专为提高会议管理效率而设计的工具，能够将预录的会议音频转录为文字，并通过先进的AI技术提取会议摘要和待办事项，一键创建或同步到指定的GitHub仓库中的Issues中。其轻量级设计和纯原生HTML实现极大地简化了部署和使用流程。
- 开箱即用，无需复杂配置环境，无后端及数据库，极致轻量化！

### 核心功能
1. **音频转文字**：任务回声能够将预录的会议音频文件转录为文字。
2. **AI摘要与待办事项提取**：通过集成的AI算法，能够从转录的文字中自动提取会议的核心摘要和待办事项。
3. **一键同步到GitHub Issues**：用户可以配置目标GitHub仓库的访问权限，将AI提取的摘要和待办事项一键同步为该仓库的Issues，方便后续跟踪和管理。
4. **极致轻量化设计**：采用纯原生HTML技术实现，无需依赖后端服务器或数据库，使得部署和使用更加灵活便捷。
5. **个性化输出**：用户可以通过编辑AI提示词来自定义输出格式，满足不同场景下的需求。

### 安装与配置

#### 1. 配置环境
- **语音转文字.py**：该脚本负责将音频文件转换为文字。使用前，请确保已安装ffmpeg工具。ffmpeg的下载地址可以在项目的release页面或[ffmpeg官网](https://ffmpeg.org/)找到。
  <img width="1788" height="332" alt="image" src="https://github.com/user-attachments/assets/ca5277e1-b4ff-4d10-a121-05fa44dd1e3d" />
- **config.conf**：配置文件用于设置AI服务的API地址和GitHub仓库的访问token。
  <img width="1084" height="430" alt="image" src="https://github.com/user-attachments/assets/5543bb33-99cc-427e-90d9-c34ab1d69130" />
  - 目前仅支持deepseek的API。
  - 提供repo的token以获取仓库访问权限。

#### 2. 运行程序
- 首先，运行语音转文字.py脚本，该脚本将处理音频文件并生成文本输出。
- 然后，打开main.html文件，进入任务回声的操作界面。
  <img width="493" height="773" alt="33ea560f1aa8c5df2507bfd0c441c3e3" src="https://github.com/user-attachments/assets/16d2fcec-e289-4ae3-8784-04e0c696ce93" />

#### 3. 添加音频文件
- 将预录的会议音频文件（支持常见音频格式）放入预先设定的监控文件夹中。
- 任务回声将自动检测文件夹中的新增音频文件，并使用ffmpeg进行转录处理，生成对应的文字文件。
- 在任务回声的Web界面中，用户可以编辑AI提示词以自定义输出格式，从而实现更符合需求的个性化输出。
- 用户可以通过选择文本文件（目前支持.txt格式）或直接拖拽文件的方式导入文本内容，系统将使用AI技术生成结构化且格式化的markdown形式的总结。
- 最后，用户可以编辑Issues的标题和内容，确认无误后，点击上传按钮，即可将会议摘要和待办事项同步至GitHub仓库中的Issues。

### 优势总结
- **高效性**：自动生成会议纪要和待办事项，节省大量时间。
- **便捷性**：纯前端设计，无需服务器配置，易于部署使用。
- **灵活性**：支持多种音频格式输入，可根据需要调整输出格式。
- **协作性**：无缝对接GitHub Issues，支持团队协作管理。
